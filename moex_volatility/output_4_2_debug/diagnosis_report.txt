========================================================================
  КЛЮЧЕВОЕ ОТКРЫТИЕ: ТРИ РАЗНЫЕ МЕТРИКИ SHARPE
========================================================================

В проекте используются ТРИ разных определения Sharpe:

| Файл                        | Метрика         | Определение                                      |
|-----------------------------|-----------------|--------------------------------------------------|
| v4_S*_daily.csv             | MeanSharpe      | Двухшаговое среднее: mean(year -> mean(tickers))  |
| v4_full_daily.csv           | Net0.40Sharpe   | Плоское среднее всех ticker-year Sharpe           |
| v4_portfolios_daily.csv     | net_sharpe      | Sharpe портфельного ряда доходностей (EW)         |

Источник кода:
- v4_tables.py -> v4_full_daily.csv (плоское среднее ~68 значений)
- strategies_walkforward_v4.py -> v4_S*_daily.csv (двухшаговое)
- v4_portfolios.py -> v4_portfolios_daily.csv (Sharpe агрегированного ряда)

Формулы:
1. MeanSharpe = mean_years(mean_tickers(Sharpe_i,y))
2. Net0.40Sharpe = mean(all Sharpe_i,y)  — все ticker-year одинаковым весом
3. EW Portfolio Sharpe = mean(r_portfolio) / std(r_portfolio) * sqrt(252)

ПРИМЕР (S2_Bollinger, подход C):
  MeanSharpe:       -0.2552  (из v4_S2_daily.csv)
  Net0.40Sharpe:    +0.4574  (из v4_full_daily.csv)
  EW Portfolio:     +2.6139  (из v4_portfolios_daily.csv)

========================================================================
  ПРОБЛЕМА 1: S2_C — отрицательный per-ticker, но EW = 2.61
========================================================================

ВЕРДИКТ: НЕ баг. Три разные метрики, все корректны.

КОРНЕВАЯ ПРИЧИНА:

1. v4_S2_daily.csv (MeanSharpe = -0.26):
   - Двухшаговая агрегация: для каждого года средний Sharpe по тикерам,
     затем средний по годам
   - Даёт равный вес каждому году
   - Некоторые year-слоты имеют сильно отрицательные значения

2. v4_full_daily.csv (Net0.40Sharpe = +0.46):
   - Плоское среднее всех ticker-year Sharpe
   - Из MERGED данных V4+V4.1 — включает ВСЕ 17 тикеров
   - Тикеры без активных сигналов C -> fallback к позициям A
     (Sharpe ~ Sharpe_A для этих тикеров)

3. EW Portfolio (Sharpe = 2.61):
   - Sharpe агрегированного временного ряда EW-портфеля
   - 17 тикеров с низкой корреляцией -> эффект диверсификации
   - Типичный ratio per-ticker -> portfolio: 4-7x

Per-ticker данные S2_C (2022-2025):
  - Только 4/17 тикеров имели отличные от A позиции:
    MOEX (+0.64), NVTK (+0.59), TATN (-0.95), LKOH (-1.04)
  - Остальные 13: режимный фильтр не нашёл улучшений -> позиции = A
  - EW-портфель S2_C = 13 тикеров A + 4 тикера C
  - Изменение 4 тикеров снижает vol портфеля -> рост Sharpe

РЕКОМЕНДАЦИЯ:
  - В паспортах использовать v4_full_daily.csv Net0.40Sharpe (плоское)
  - В сводной таблице 4.6 — EW Portfolio Sharpe (портфельное)
  - В тексте чётко различать метрики
